Abstract

Smart Energy Systems (SES) is critical in ensuring conservation of energy of homes and offices. The system is designed to detect human presence in a room and turn on the lighting system. Incase the temperature is high, the system also turns on the air conditioning system. This system has hybrid capabilites, as the user can manually turn on, or turn off either of these systems via a mobile app. The mobile app also has a dashboard for the user to view the metrics in energy consumption in the room. This report presents the development and implementation of this smart energy system.

The proposed system uses an ESP32 to control a bulb and a fan based on room occupacy and temperature. An Ultra sonic sensor detects human presence, triggering an LED bulb via a relay module. A temperature sensor monitors room conditions, activating a fan if the room is occupied and temperature exceeds the pre-determined unit(eg 25 degrees). If no motion is detected for some time (eg 3 minutes), the ESP32 switches off both components to save energy.

Introduction

Smart Energy Systems is a smart solution designed to enhance conservation of energy in homes and workplaces. In this document, we will talk about the problems that this system aims to solve, the main goals of the project, and the key features needed for it to work effectively.

Problem Statement

In Uganda, where only 22% of the population has access to electricity, energy conservation is critical, particularly in community spaces that rely on costly grid or generator power. Unnecessary lighting, air conditioning, etc in unoccupied rooms wastes limited energy resources, increasing costs and limiting access to power for other essential uses. This project addresses SDG 7 (Affordable and Clean Energy) by reducing energy waste, promoting sustainable energy use in resource-constrained settings.

Project Goals

Develop Real-Time Detection: To create a system that accurately identifies occupancy in a room using an Ultra sonic sensor for real time detection

Implement Real-Time Mechanisms: To ensure the system promptly activates an LED bulb and fan based on occupancy and temperature conditions

Enhance Energy Efficiency: To design the system to minimize energy waste by switching off components when no motion is detected for some time eg 3 minutes

Provide Continuous Status Updates: To incorporate a mobile app to display real-time status of light, fan and temperature

Create a User Management Interface: To develop an admin interface to be managed by authorized personnel in order to review the energy consumption hence aid in decision making

Functional Requirements

Allows Human Presence Detection and Triggering: The Ultra sonic sensor allows detection of human presence in the premises and triggers the LED bulb via a relay module

Allows Temperature Monitoring and Response: The temperature sensor monitors room conditions, activating the fan if both temperature exceeds pre-determined conditions (eg 25 degrees) and occupacy is detected

Allows Status Display: The mobile app displays real-time status, including light/fan on/off and current temperature

Allows Energy Saving Automation: The system switches off the bulb and fan if no human presence is detected in order to conserve energy

Limitations of SES

1. Power Supply Constraints:
The system relies on a stable power source for the components. Power outages and fluctuations disrupt the functionality of the system

2. Fixed Thresholds:
The temperature threshold and room inoccupancy timeout are hardcoded, lacking flexibility for different environments or user preferences

3. Environmental Factors:
The system does not account for humidity and air quality, which could be relevant for ventilation in community spaces

Future Enhancement

1. Adaptive Control Logic
Using Machine Learning to predict occupancy patterns and optimize energy saving settings based on historical data

2. Power Management
Integrating battery backup or solar power to ensure functionality during power outages which is critical for community spaces

3.Scalability and Modularity:
Creating a modular framework to easily add or remove sensors (eg additional lights or fans) without major rewiring.
